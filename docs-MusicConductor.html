MusicConductor

Class

A controller designed to play multiple of it's children simultaneously (for instance, two layers of the same song). Provides an interface for tracks to be cued in and out of the arrangement.

Extends: MultiTrackPlayer

Properties

  • GcNode LeadTrack

    The track from which all others take their timing cues. When a new track is cued in or played, they play from the same position as the lead track. When this track finishes and loops, all others restart regardless of whether they have reached the end or not (in practice, the difference in time is unlikely to be more than a few milliseconds, but this could add up to a noticeable lag over multiple repeats). This is automatically and dynamically assigned, so does not need to be handled manually. Description of the method. Note that publicly, this property is read only, and cannot be reassigned externally.

Methods

  • Boolean Cue(GcNode track)

    Toggles the given child track in or out of the arrangement. Returns true if the track is found and cued in/out.
    Returns: true if the track is found and cued successfully.

  • Boolean CueIn(GcNode track)

    Cues the given child track in if and only if it is not playing, otherwise it does nothing.
    Returns: true if the track is found and cued successfully. False if the track was already playing.

  • Boolean CueOut(GcNode track)

    Cues the given child track out if and only if it is playing, otherwise it does nothing.
    Returns: true if the track is found and cued successfully. False if the track not was already playing.

  • Also:

    Boolean CueInIdx(int idx), boolean CueOutIdx(int idx), boolean CueInName(string name), and boolean CueOutName(string name). The usage and effects of these methods should be easily inferable, but please feedback if this is not the case!
